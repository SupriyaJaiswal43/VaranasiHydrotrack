Map Integration

The project uses Leaflet.js, an open-source JavaScript library, to create interactive maps. Leaflet makes it easy to display maps and add various layers and markers. The map serves as the main interface for users to explore data, providing visual representations of different water quality parameters.

Base Layers

Base layers are the foundational map tiles that provide the basic visual representation of the map. The project includes different base layers, such as:

  • Street Map: A standard map view showing roads and pathways.
  • Satellite View: Aerial imagery providing a real-world view of the area.
  • Terrain View: A map highlighting geographical features like hills and valleys.
These base layers can be switched according to user preference, allowing them to view the map in different styles.

Data Layers

  • Markers: These are points on the map representing specific water quality measurement locations. They are color-coded based on the Water Quality Index (WQI), with colors like green for good quality, yellow for moderate, and red for poor quality. Clicking on a marker displays detailed water quality information.
  • Heatmap: This layer shows the density or intensity of water quality measurements, helping to identify areas with high or low concentrations of data.
  • Cluster Groups: This feature groups nearby markers together to reduce map clutter. When zoomed out, it prevents the map from being overcrowded with markers, making it easier to navigate.

Controls and Interactivity

To enhance user experience, the project includes interactive controls:

  • Checkboxes: Allow users to toggle the visibility of different layers (e.g., markers, heatmap, clusters).
  • Dropdown Menu: Lets users quickly navigate to specific locations by selecting them from a list, zooming the map to the chosen area.
  • Download Button: Offers the option to download the water quality data in CSV format for further analysis or record-keeping.

Data Processing and Display

Water quality data is fetched from a backend source and processed for display on the map. The data includes parameters like pH, TDS, Turbidity, and other chemical indicators. The project calculates the WQI and determines marker colors accordingly, providing a quick visual assessment of water quality in different areas.

Instructions and User Guidance

The project includes a section that guides users on how to interact with the map and interpret the data. This guidance covers how to use map controls, understand marker color coding, and utilize other features.

Technology Stack

The technology stack for this project includes:

  • HTML/CSS/JavaScript: For frontend design, user interface, and map rendering.
  • Leaflet.js: For map functionalities and interactions.
  • MongoDB: For storing water quality data.
  • Node.js: For server-side processing and API handling.
